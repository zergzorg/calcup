# Sport And Clothing Calculators Roadmap

Status: active
Created: 2026-04-25
Scope: продуктовая дорожная карта разделов `/sport` и `/clothing`

## Overview

Добавить в Calcup две новые линейки калькуляторов:

- `Спорт` — быстрые расчёты для бега, тренировок, пульса, темпа, скорости и дистанций.
- `Одежда и обувь` — конвертеры размеров кроссовок, обуви, одежды и популярных брендов.

Первый шаг уже должен быть реестровым: категории и заглушечные карточки появляются в навигации, но конкретные калькуляторы остаются `soon` до реализации формул, UX, тестов и SEO.

## Context

Вдохновение по спорту взято из мобильных экранов с такими сценариями:

- пульсовые зоны;
- метроном / каденс;
- дистанция / темп / время;
- темп / скорость;
- размер кроссовок как отдельный обувной сценарий.

Для одежды ключевой первый сценарий — кроссовки и обувь: перевод между `US`, `UK`, `RU`, `EUR`, `CM`, полом и брендами вроде `Asics`, `HOKA`, `Nike`, `Adidas`.

## Development Approach

Новые калькуляторы делать по существующему контракту:

- регистрировать в `src/data/calculators.ts`;
- для готовых калькуляторов создавать feature-модуль в `src/features/<slug>/`;
- визуальную форму строить через `src/features/calculator-design-system.css`;
- не добавлять локальные переопределения рамок, радиусов, высот контролов, focus-ring и result-panel layout;
- чистые расчёты держать в `lib/calculations.ts`;
- интерактивное состояние держать в composable, если форма становится сложнее одного компонента.

## UX Notes

### Sport

- Для `pace-speed` нужен максимально короткий экран: слева темп, справа скорость, с крупным результатом.
- Для `distance-pace-time` нужны быстрые чипы дистанций: `3`, `5`, `10`, `15`, `21.1`, `42.2` км.
- Для `heart-rate-zones` нужен цветной список зон, но внутри Calcup его стоит адаптировать под общий calculator design system, а не копировать мобильный UI.
- Для `metronome` нужен отдельный вопрос по аудио: можно ли использовать Web Audio API и как сохранять пользовательский темп.

### Clothing

- Для `sneaker-size` важна таблица результатов по форматам `US`, `UK`, `RU`, `EUR`, `CM`.
- Пол и бренд должны быть явными переключателями / select, потому что размерные сетки меняются.
- Брендовые таблицы должны иметь версию источника и дату актуальности.
- Нужно избегать обещания идеальной посадки: размер зависит от модели, полноты стопы и производителя.

## Data Strategy

- Базовые таблицы размеров хранить как локальные typed data.
- Для брендов начать с ограниченного набора: `Nike`, `Adidas`, `Asics`, `HOKA`, `New Balance`, `Puma`.
- Для каждого бренда фиксировать:
  - пол / линейку;
  - систему размеров;
  - длину стопы / CM;
  - дату проверки;
  - ссылку на источник в комментарии или отдельной reference-структуре.
- Не смешивать мужские и женские сетки без явной нормализации.

## Testing Strategy

Для каждого ready-калькулятора:

- unit-тесты на формулы и граничные значения;
- `npm run type-check`;
- `npm run test`;
- `npm run build`;
- ручной smoke на desktop и mobile `360px`;
- проверка, что `soon`-калькуляторы не попадают в поиск готовых инструментов и не индексируются как готовые страницы.

## Risks / Open Questions

- Брендовые размерные сетки меняются и могут отличаться по модели. Нужны источники и дата актуальности.
- `heart-rate-zones` связан со здоровьем: нужен справочный дисклеймер и аккуратные формулы.
- Метроном требует аудио-разрешений браузера и проверки поведения на мобильных.
- Нужно решить, добавлять ли готовые спортивные калькуляторы также в `health` как связанные, или держать их только внутри `/sport`.

## Post-Completion

После первого ready-калькулятора в каждом разделе:

- обновить `public/sitemap.xml` только для категорий/страниц с готовым содержимым;
- проверить SEO title/description через `src/composables/useSeo.ts`;
- проверить карточки на главной и в футере;
- пройти mobile smoke для списка категории и конкретного калькулятора.
