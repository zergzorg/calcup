# Кредитный калькулятор с графиком платежей и печатной формой

Status: implemented  
Created: 2026-04-24  
Scope: добавить в `calcup` кредитный калькулятор с расчетным ядром, UI, досрочными погашениями, графиком платежей и отдельной печатной A4-формой

## Overview

Нужно реализовать самостоятельный кредитный калькулятор для сайта Calcup. Пользователь должен вводить условия кредита, видеть основные итоги, добавлять досрочные погашения, выбирать стратегию досрочного погашения, просматривать график платежей и получать отдельную печатную форму с логотипом, водяным знаком и расчетом.

Работа должна идти строго итерациями: анализ и планирование → согласование → реализация этапа → проверка → согласование следующего этапа. Новые PDF-зависимости нельзя добавлять без отдельного согласования.

## Context

Текущее состояние проекта по первичному осмотру:

- Стек: Vue 3, TypeScript, Vite, Vue I18n.
- Роутер в проекте не найден: приложение сейчас собрано вокруг `App.vue` и виджетов рабочего стола.
- Предпочтительный пользовательский путь для калькулятора: `https://calcup.ru/credit-calc/`.
- На экране `credit-calc` должен оставаться фон основного рабочего стола Calcup, но вместо набора виджетов показывается только кредитный калькулятор.
- Тестового раннера в `package.json` нет; доступная автоматическая проверка сейчас: `npm run build` (`vue-tsc -b && vite build`).
- PDF-библиотек в зависимостях нет.
- Print/PDF-механика в проекте не обнаружена.
- SEO уже частично настроено: есть `src/composables/useSeo.ts`, мета-теги и JSON-LD в `index.html`, `public/sitemap.xml`, `public/robots.txt`, `public/og-image.jpg`.
- Логотип для печатной формы можно брать из `public/calcup.svg`.
- Основные UI-компоненты проекта сейчас виджетные: `TimerWidget`, `TaskPlanner`, `SoundMachine`, `DateTimerWidget`, `SnakeGame`, `DesktopLayout`, `SettingsMenu`, `LanguageSwitcher`.
- Готовых универсальных компонентов формы, таблицы, модального окна, money/date-форматтеров для кредитного калькулятора не найдено.
- В проекте уже есть `src/composables/useMobileLayout.ts`, `src/style.css` и мобильный план в `docs/plans/active/2026-04-24-mobile-responsive-refactor.md`; при UI-этапах нужно учитывать возможные пересечения с мобильной адаптацией рабочего стола.

## Development Approach

Основной принцип: сначала создать чистую расчетную логику без UI, затем подключать ее к интерфейсу и печатной форме.

Предлагаемая структура фичи:

```txt
src/
  features/
    credit-calculator/
      components/
        CreditCalculatorView.vue
        CreditLoanForm.vue
        CreditResults.vue
        CreditEarlyRepayments.vue
        CreditScheduleTable.vue
        CreditPrintView.vue
        CreditPrintActions.vue
      lib/
        annuity.ts
        calculateCreditSchedule.ts
        date.ts
        money.ts
        validation.ts
      types/
        credit.ts
```

Предпочтительное размещение: отдельный экран `calcup.ru/credit-calc/`, который переиспользует визуальный фон и базовую атмосферу основного рабочего стола, но не рендерит остальные виджеты. На этом экране должен быть только кредитный калькулятор и связанные с ним действия.

Так как роутера в проекте пока не найдено, на этапе 1 нужно выбрать легкий способ маршрутизации под Vite/static hosting: встроенный Vue Router, минимальный path-switch в `App.vue`, либо другой вариант, совместимый с деплоем на `calcup.ru/credit-calc/`. Без отдельной необходимости не усложнять routing.

MVP PDF:

- сначала качественная печатная форма и кнопка печати через `window.print()`;
- сохранение в PDF через системный print dialog;
- прямой PDF-экспорт только после отдельного согласования подхода и зависимости.

## Testing Strategy

Минимальная автоматическая проверка на каждом кодовом этапе:

- `npm run build`.

Для расчетного ядра нужно отдельно согласовать тестовый раннер, потому что сейчас в проекте нет `vitest`/`jest`/другой тестовой инфраструктуры. Предпочтительный вариант для Vue + Vite: `vitest`, но зависимость добавлять только после согласования.

Ручная проверка UI:

- desktop viewport;
- mobile viewport;
- корректность форматирования денег и дат;
- отсутствие горизонтального переполнения;
- корректность печатной формы на A4;
- отсутствие интерактивных элементов в печати.

## Technical Notes

- Не копировать дизайн, тексты, верстку, CSS, HTML или брендовые элементы сторонних сайтов.
- Расчетная логика не должна зависеть от Vue-компонентов.
- Текст водяного знака и путь логотипа не должны находиться внутри расчетной логики.
- Первый релиз поддерживает только `PaymentType = 'annuity'`.
- Деньги и даты форматировать через локальные утилиты фичи, если в проекте не появятся общие форматтеры.
- Кириллица должна корректно отображаться в UI, печати и PDF/print dialog.
- SEO-тексты должны быть полезными для человека и соответствовать реальному функционалу страницы; не делать keyword stuffing и не обещать банковски точный расчет.
- Добавление `vitest`, `@vue/test-utils`, `html2pdf.js`, `jspdf`, `html2canvas` или других зависимостей требует отдельного согласования.
